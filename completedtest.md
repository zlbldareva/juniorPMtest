# Анализ проекта и план действий

## 1. Идентификация проблемы

- <u>**План-графика работ нет (либо он не актуален):**</u> финальная цель (оплата подписки) и текущие действия команды (рефакторинг, замена иконки и проч.) не связаны единым планом.

- <u>**Отсутствует приоритезация:**</u> от СЕО поступают разные масштабные задачи, но не одна не связана с тем, что база данных постоянно падает. Почему он дает задачу переделать иконку, если приложение в принципе может не быть работоспособным?

- <u>**Отсутствует MVP:**</u> границы проекта размываются из-за добавления новых крупных функций (раздел с питанием и рецептами) поверх незавершенного базового функционала.

- <u>**Коммуникация слабо налажена:**</u> один из сотрудников не отвечает неделю, дизайнеру не выдали исходные данные.

- <u>**Нет прозрачности статуса работы:**</u> формальный статус «В работе» (по тестированию) скрывает реальные проблемы (падение БД).

## 2. Выбор методологии и первые шаги

Я бы предложила ввести **Scrum**, гибкая методология, предполагающая беклог (постоянно обновляющийся документ с расставленной приоритезацией).  
Введем спринты длительностью 1-2 недели, для каждого спринта будем определять цель и порядок ее достижения. Введем дейлики, наладим коммуникацию внутри команды, обеспечим оперативную работу с возникающими проблемами и прозрачность прогресса работы.

На первой встрече с командой я бы:

- выяснила MVP проекта, составила беклог проекта, узнала что уже в работе, на какой стадии, какая проблематика;
- обсудила предлагаемое решение (Scrum, дейлики);
- определила цели и задачи на первый спринт, составила беклог спринта, назначила ответственных, назначила дату следующей встречи в начале следующего спринта.

### БЕКЛОГ ПРОЕКТА:

- определить и утвердить MVP для релиза к выставке «Спорт-2025»;
- восстановить и протестировать работу базы данных;
- разработать и передать Android-разработчику стабильное API;
- согласовать контент-план для раздела «Тренировки»;
- определить MVP для модуля «Оплата подписки».

## 3. Коммуникация со стейкхолдером

**Тема:** План действий по монетизации и развитию приложения «Фитнес-гуру»

Уважаемый, [Имя Отчество]!

Я полностью разделяю Вашу озабоченность по поводу монетизации и важности запуска оплаты подписки для инвесторов.

На настоящий момент мы с командой фокусируемся на устранении критических технических проблем (в частности, нестабильности базы данных), которые напрямую влияют на работоспособность приложения. Без этого любые новые функции, включая оплату, рискуют работать некорректно.

На этой неделе мы завершаем диагностику и исправление проблем с базой данных. Как только это будет выполнено, мы сможем гарантировать стабильную работу системы.

Следующим шагом станет интеграция оплаты подписки. Мы уже начали проработку технических требований и оценку сроков. Я подготовлю для вас детальный план по реализации к концу недели с учетом квартальных сроков.

Задача об изменении иконки приложения добавлена в бэклог проекта. Мы с командой обсудим её в рамках планирования следующего спринта.
